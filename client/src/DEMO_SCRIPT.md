## ⚠️ COMMON QUESTIONS & ANSWERS

**Q: Is the dual pricing automatic or manual?**  
A: Completely automatic. Buyers never select pricing mode. The system determines it based on quantity and seller-defined thresholds.

**Q: How do sellers set their pricing?**  
A: Through the Add Product page, sellers input: retail price, wholesale price, and the quantity threshold. The system handles the rest.

**Q: Can different products have different thresholds?**  
A: Yes, each product has its own threshold. One product might be 10 units, another might be 5.

**Q: What if a buyer mixes products in cart?**  
A: Each product is evaluated independently. If Product A reaches threshold, it gets wholesale pricing. Product B is priced separately.

**Q: How do you prevent sellers from setting unrealistic prices?**  
A: Admin approval workflow. All products must be approved before going live.

**Q: Is this mobile-responsive?**  
A: Yes, fully responsive design. (Demo by resizing browser)

**Q: What about real payment processing?**  
A: This prototype demonstrates the flow. Real implementation would integrate payment gateway APIs.

**Q: How does multilingual work?**  
A: Toggle in navigation. Sellers input product info in both languages. UI elements are translated.
